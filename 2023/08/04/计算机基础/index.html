计算机基础

HTTP响应码

1xx：服务器收到信息，需要继续操作
2xx：成功
3xx：重定向
4xx：客户端错误/请求错误
5xx：服务端错误/响应超时

HTTP请求方式

1.GET 2.POST 3.PUT 4.PATCH 5.DELETE 6.COPY 7.HEAD 8.OPTIONS 9.LINK
10.UNLINK 11.PURGE 12.LOCK 13.UNLOCK 14.PROPFIND 15.VIEW

数据包结构

请求包结构

第一部分：请求行（数据包中的一行内容）
请求行包括三部分内容：1.请求方式（GET/POST）
2.请求资源路径（端口号之后的内容，比如、/appname/servlet)
3.协议的类型与版本

第二部分：若干消息头（消息头是由w3c定义的一些有特殊含义的键值对）
消息头的样式，比如：connect-type-text/html;
服务器和浏览器都会遵循这些消息头的约定。
消息头一般由服务器或者浏览器自动生成，但是也可以通过编程的方式生成

第三部分：空行

第四部分：实体内容
如果是post请求方式，请求参数及值会放在这
如果是get请求方式，请求参数和值是包含在请求资源路径里面

响应包结构

第一部分：状态行，协议的类型与版本，状态码

第二部分：若干消息头

第三部分：实体内容，服务器返回给浏览器的处理结果

linux操作系统基本命令

cd / 切换到根目录
cd /root 切换到根目录下的root目录
cd .. 切换到上一级目录
cd ~ 切换到home目录
cd - 切换到上次访问的目录

ls 查看当前目录下的所有目录和文件
ls -a 查看当前目录下的所有目录和文件（包括隐藏的文件）
ls -l 或ll 查看当前目录下的所有目录和文件（以列表方式显示查看，显示更多信息）

windows操作系统基本命令

命令行

cd 进入文件夹
cd /d d:\GAME\ 将当前目录切换为d盘的GAME文件夹
cd /d c:\USER\ 回到原用户目录
cd.. 跳至上一级目录
cd /? 查询命令cd的用法

cls 清楚当前那屏幕内容
dir 显示当前目录下的子目录与文件的列表
md 创建新的目录
rd 删除空的文件夹
type 浏览一个文件的内容
ipconfig 查看当前主机的网络配置
del 删除文件

基础操作命令

ctrl+c/v 复制/粘贴
alt+tab 切屏
ctrl+alt+a 截图
ctrl+alt+s 录屏
caplocks 开启/关闭大小写
fn+esc 开启/关闭f1-f12功能键
win+r 打开运行窗口
